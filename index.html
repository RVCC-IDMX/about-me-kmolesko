CSS Animation:

Animation is a great way to draw the user’s attention to certain content on a webpage or to simply increase a site’s visual or artistic impact. In CSS, animation is a styling technique that allows the properties of an element to change over a select period of time. This can create a plentitude of different visual effects.

In order to implement CSS animation, the developer must initially specify the styling they wish their element to take on during the animation sequence. This is done within the brackets of an “at-rule” called the @keyframes rule, which allows the developer to define properties for multiple waypoints, or stopping points, throughout the animation. .

Keyframes allow for simple transition from one style to a second using the syntax “from” and “to,” but it also allows for the use of percentages where in the process of an animation the styles will change. Percentages also allow many waypoints to be defined. After calling upon the @keyframes rule, the animation name assigned to the element that will take on these properties is inputted, followed by brackets.

@keyframes example { 0% {background-color: red;} 25% {background-color: yellow;} 50% {background-color: blue;} 100% {background-color: green;} }

The above syntax indicates to CSS that the background color of the element with the animation name “example” will change from red to yellow to blue to green. When the animation is 0% complete (the initial property) the background will be red. After 25% of the animation time has passed, it will change to yellow, and so on.

@Keyframes is not the only property involved in animation- some coding must be done within the animated element. Most crucially, the animation must be give a name using the animation-name property, which is the name that will be typed next to the keyframes rule.

In addition to @keyframes and animation-name, there is one more required property in order to get an animation running: this is animation-duration. The default value for this property is 0 seconds, so a time value must be specified in order for the animation to work. Using only animation-duration, animation-name and @keyframes, animations can be created with other default settings applied. However, there are a variety of additional animation properties that can be used.

Animation-delay specifies a delayed animation start. Negative numbers may also be used to start the animation as if it had been running for the specified number value’s amount of time.

By default, animations run one time and do not repeat. This can be changed with the property animation-iteration-count, which can indicate a number value that indicates how many times the animaton will run. To repeatedly run the animation, the value “infinite” is used.

Animations also run forwards by default, but with the animation-direction property they can be run forwards, backwards, or in alternate cycles using values normal, reverse, alternate, or alternate-reverse.

The animation-timing-function specifies the speed curve of the animation. The default is ease (slow start, then fast, then slow end), but other values include linear (constant speed), ease-in (slow start), ease out (slow end), or cubic-bezier(n,n,n,n) which allows you to define your own cubic function using chosen values.

If an animation is not infinite, the element will have a certain styling before and after the animation is played. A developer may use animation-fill-mode to specify the style while the animation is not playing. The default value is none, which calls for no styling applied before and after the execution of the animation. If value forwards is applied, the element retains the style set in the last keyframe after the animation’s end. If backwards is applied, the element takes on the style set by the first keyframe prior to the animation’s start (if a delay is present). With the value both, the element starts off with the styling of the first keyframe, and ends with the styling of the last.

To simplify the use of these properties, the shorthand property animation combines many of them into one, as exemplified below:

div { animation: animation-name animaton-duration animation-timing-function animaton-delay animation-iteration-count animation-direction; }
